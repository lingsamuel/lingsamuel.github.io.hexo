Environment Maintenance Guide

发表于 2018-08-06 | 更新于 2018-08-08 | 分类于 日常 |
本文字数： 1.4k | 阅读时长 ≈ 1 分钟

前略，天国的 SSD。

又要重装系统了，数据又丢了，设置又没了，痛苦。我决心要弄个什么东西来做同步。

头疼的是配置都是小问题，但是丢数据是真的难过。

Environment Setup/Sync Guide

Windows

1. Change Location of Windows

这次重大损失就损失在我的桌面文件夹……Win10 是自己提供了功能来修改位置的（资源管理器右键属性），但是可能我忘记改了，Documents Downloads 之类的目录我倒是改了，唯独 Desktop 没有，SSD 一挂损失惨重。

6. 字体

同步配置

VS Code configuration

{UserDir}\AppData\Roaming\Code\User

Jetbrains IDEs configuration

可能需要手动导出成 .jar

.ssh

一般来讲我不会备份秘钥，因为我没有限制访问级别的服务器……最坏情况也可以通过服务商提供的终端访问。

但是 config 还是可以备份一下的，主要是配置了代理设置。

git.config

全局的 {UserDir}.gitconfig 和 {UserDir}.gitignore 配置。

同步方式

配置一般不是很大，弄了个 GitHub Private Repo 来存。

然后在 repo 目录下建一个 sync-files.txt，里面存路径就成了。需要注意这个 txt 得是 LF 的。

同步数据

数据同步比较头疼，因为太大了，放 Git 显然不合适。OneDrive 和 GoogleDrive 也许是个更好的选择，一些不怎么变的数据甚至可以放网盘。

Manic Time database

需要手动导出，可以自己建服务器，但是太麻烦了。
