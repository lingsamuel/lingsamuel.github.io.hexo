发表于 2016-12-22

Introduction to basic Kotlin.

What’s Kotlin

Kotlin 是 JetBrains 推出的基于 JVM 的新一代的静态类型编程语言。

一大强推的 feature 是对 Java 的 100% 兼容，并且有 toJs 的工具，就像他们自己的简介一样，

Statically typed programming language for the JVM, Android and the browser.
100% interoperable with Java™

Why Kotlin

给出我自己的理由。

  1. 表达力强。
  2. 100% 兼容 Java。
  3. 有函数式特性。
  4. 避免了一些 Java 的坑。
  5. Null-Safety

第一点是非常明显的。作为入门语言是 Java 的人，自我了解了更多的语言（尤其是函数式的语言后），愈发难以忍受 Java 的复杂。
Java 是一门非常非常咸的语言，初学者很难掌握 OOP 的精髓（当然也是我个人理解的），这会导致复杂度的急剧上升（我也有过类似经验），出现那种“为了 OO 而 OO”的代码，是非常难看并且难以维护的。
而 Kotlin 有很多的糖，语法表达力也是足够的，能够使代码更加简洁、清晰，易于维护。

第二点是最重要的一点，也是使用 Kotlin 的重要理由之一。它能完美兼容 Java，这意味着你可以慢慢迁移到 Kotlin，不顺手了可以立马切回 Java 而不用担心任何问题。并且使得 Kotlin 天生就拥有数量庞大的库。

第三点纯粹是写得爽。

第四点参考 Comparison to Java

第五点似乎没那么紧要，但是 Maybe 式的 Null-Safety 强制作者考虑 null 情形，大部分情况下是有利的，而且因为第一点，这并不会使得你的代码长很多。

额外加分项：

Kotlin 不是 Haskell 那样纯学术内核的，而是和工程紧密结合的。

Kotlin 的作者们是世界上最强的一帮静态分析专家和工程师，我相信他们的经验 XD。

我是 JetBrains 粉丝。

Quickstart (Basic)

semicolon free

var a = 1; var a = 1

package

package a.b

不需要严格遵循文件夹

function

fun func(a: Int): Int { } fun sum(a: Int, b: Int) = a + b

string
